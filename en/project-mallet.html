CLIENT CONTEXT

Fondation Mallet is a medico-social foundation based in the Yvelines department (78), operating across 11 establishments spread over 3 geographic sites. It supports people with disabilities (intellectual impairment, multiple disabilities, autism spectrum disorders) and employs hundreds of healthcare professionals, educators and administrative staff.

Its ageing and heterogeneous IT infrastructure no longer met the requirements for security, traceability and centralised management. The foundation therefore commissioned our team to carry out a full audit of its information system, followed by the deployment of a modernised infrastructure based on Windows Server 2025 and Active Directory.

PROJECT DESCRIPTION

The Mallet Project is a systems infrastructure project carried out as part of the BTS SIO programme (Year 1). It is structured around 4 progressive missions:

Mission 1 — Infrastructure audit: analysis of the existing setup (hardware inventory, network, services), identification of vulnerabilities and production of a full audit report with recommendations.

Mission 2 — Active Directory deployment: installation and configuration of the Windows Server 2025 domain controller, creation of 3 OUs (ADMIN, PÉDAGOGIQUE, DIRECTION), setup of user accounts and security groups.

Mission 3 — Group Policy Objects (GPO): defining and deploying per-OU policies — access restrictions, encryption, web filtering and hardening of Windows 11 Pro client workstations.

Mission 4 — Supporting services deployment: GLPI (asset management & ticketing), Zabbix (network monitoring), Nextcloud (secure file sharing), PaperCut MF (print management).

MISSION 1 — INFRASTRUCTURE AUDIT (VALIDATED)

The first mission, fully completed and validated, involved producing a comprehensive inventory and assessment of Fondation Mallet's IT infrastructure.

Audit scope:
▸ Hardware inventory — full census of workstations, servers and network equipment
▸ Network architecture — topology, segmentation, interconnection of the 3 sites
▸ Existing services — directory (LDAP), messaging, file shares, print management
▸ Security — access rights analysis, patch management review, practices assessment
▸ Compliance — verification of key ANSSI and GDPR requirements

Key vulnerabilities identified:
▸ No centralised domain controller → uncontrolled account management
▸ No OU-based network segmentation → all users share equivalent rights
▸ No active monitoring (no alerting on failures or incidents)
▸ No ticketing system → IT interventions untracked
▸ Unsecured, non-centrally-backed-up file shares

Adopted recommendations:
▸ Deploy a Windows Server 2025 domain controller with Active Directory (domain mallet.local)
▸ Structure into 3 OUs: ADMIN, PÉDAGOGIQUE, DIRECTION
▸ Deploy GLPI for asset management and helpdesk ticketing
▸ Set up Zabbix for network monitoring and alerting
▸ Deploy Nextcloud for secure internal file sharing
▸ Integrate PaperCut MF for print management

TARGET ARCHITECTURE

The target infrastructure is built around an Active Directory domain mallet.local managed by a Windows Server 2025 domain controller. Three Organisational Units (OUs) structure access management:

OU ADMIN — System administrators, full access rights, administration tools and rights management GPO.

OU PÉDAGOGIQUE — Teachers and educators, restricted access to educational resources, web browsing filter applied via GPO.

OU DIRECTION — Directors and HR staff, access to confidential data, data encryption GPO and external device restrictions.
